1. 时间转换工具

功能介绍

时间转换工具支持时区转换、时间戳转换，并能自动识别夏令时，帮助您轻松处理不同时区的时间问题。

使用场景

  • 跨时区会议时间安排
  • Unix时间戳与人类可读时间的转换
  • 不同时区之间的时间对照
  • 夏令时自动计算

使用方法

时区转换

  1. 选择源时区和目标时区
  2. 输入需要转换的时间
  3. 点击”转换”按钮即可得到结果
  4. 系统会自动识别并处理夏令时

时间戳转换

  1. 时间戳转日期：

    • 输入Unix时间戳（秒或毫秒）
    • 选择要显示的时区
    • 点击转换获得可读日期时间

  2. 日期转时间戳：

    • 选择或输入日期和时间
    • 选择时区
    • 点击转换获得Unix时间戳

技巧提示

  • 时间戳支持10位（秒）和13位（毫秒）格式
  • 支持常见时区快速选择
  • 可以保存常用时区配置

4. 贷款利息计算工具

功能介绍

专业的贷款计算器，支持等额本息、等额本金两种还款方式，可计算组合贷款，并生成详细的还款计划表。

使用场景

  • 房贷计算
  • 车贷计算
  • 消费贷款规划
  • 还款方式对比

使用方法

基础设置

  1. 输入贷款金额：贷款本金总额
  2. 选择贷款期限：年数或月数
  3. 输入贷款利率：年利率（%）
  4. 选择还款方式：
    • 等额本息：每月还款额固定
    • 等额本金：每月还款本金固定

等额本息

  • 特点：每月还款金额相同
  • 适用：收入稳定的工薪族
  • 计算：月还款额 = [贷款本金×月利率×(1+月利率)^还款月数] / [(1+月利率)^还款月数-1]

等额本金

  • 特点：每月还款本金相同，利息递减
  • 适用：前期还款能力较强的借款人
  • 计算：
    • 月还本金 = 贷款本金 / 还款月数
    • 月还利息 = (本金 - 已归还本金累计额) × 月利率

组合贷款

  1. 分别输入商业贷款和公积金贷款金额
  2. 分别设置对应的利率
  3. 系统自动计算总还款额

查看还款计划

  • 每月还款明细
  • 本金和利息分布
  • 剩余本金
  • 总利息支出

技巧提示

  • 可对比两种还款方式的总利息
  • 支持提前还款计算
  • 可导出还款计划表

8. 加解密工具

功能介绍

专业的加密工具，支持对称加密、非对称加密、HMAC签名等多种加密算法。

使用场景

  • 数据加密存储
  • 密码加密
  • 数字签名验证
  • API签名计算
  • 敏感信息保护

使用方法

对称加密（AES/DES/3DES）

AES加密（推荐）

  1. 选择AES算法
  2. 选择模式（ECB、CBC、CTR、GCM等）
  3. 输入密钥（Key）
  4. 输入IV向量（CBC/CTR/GCM模式需要）
  5. 输入要加密的文本
  6. 点击”加密”获得密文

AES解密

  1. 使用相同的密钥和IV
  2. 输入密文
  3. 点击”解密”获得原文

非对称加密（RSA）

RSA加密

  1. 生成或导入RSA密钥对
  2. 使用公钥加密数据
  3. 使用私钥解密数据

数字签名

  1. 使用私钥对数据签名
  2. 使用公钥验证签名

Hash算法

支持多种Hash算法：

  • MD5：128位（已不推荐用于安全场景）
  • SHA-1：160位（已不推荐用于安全场景）
  • SHA-256：256位（推荐）
  • SHA-512：512位（高安全）

使用方法：

  1. 选择Hash算法
  2. 输入文本
  3. 点击”计算Hash”
  4. 获得Hash值

HMAC签名

  1. 选择HMAC算法（HMAC-MD5、HMAC-SHA256等）
  2. 输入密钥（Key）
  3. 输入要签名的数据
  4. 点击”生成签名”
  5. 获得HMAC签名值

加密模式说明

  • ECB模式：简单但不推荐，相同明文产生相同密文
  • CBC模式：需要IV，更安全，广泛使用
  • CTR模式：流加密模式，可并行处理
  • GCM模式：提供加密和认证，最安全

技巧提示

  • AES-256-GCM是最安全的对称加密方式
  • 密钥长度：AES支持128/192/256位
  • IV向量必须随机生成，不可重复使用
  • 保存好密钥和IV，丢失将无法解密

9. JWT解析工具

功能介绍

解析和验证JSON Web Token，支持Header、Payload和Signature分析，是Web开发者的必备工具。

使用场景

  • JWT令牌解析
  • Token有效期验证
  • Token签名验证
  • 调试认证问题

使用方法

JWT结构

JWT由三部分组成，用.分隔：

解析JWT

  1. 粘贴JWT令牌到输入框
  2. 系统自动解析并显示：
    • Header：算法和令牌类型
    • Payload：载荷数据
    • Signature：签名信息

查看Header

  • alg：签名算法（HS256、RS256等）
  • typ：令牌类型

查看Payload

常见字段：

  • sub：主题（用户ID）
  • iat：签发时间
  • exp：过期时间
  • iss：签发者
  • aud：接收者

验证签名

  1. 输入密钥（Secret）
  2. 点击”验证签名”
  3. 系统显示签名是否有效

生成JWT

  1. 输入Header内容
  2. 输入Payload内容
  3. 输入密钥
  4. 选择算法
  5. 点击”生成JWT”

技巧提示

  • JWT令牌不加密，只是Base64编码
  • 不要在Payload中存储敏感信息
  • 注意检查过期时间（exp）
  • 签名验证需要正确的密钥

  • $ - 字符串结束
  • \b - 单词边界
  • \B - 非单词边界

分组

  • (...) - 捕获组
  • (?:...) - 非捕获组
  • (?=...) - 正向前瞻
  • (?!...) - 负向前瞻
