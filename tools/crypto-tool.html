📘 AES加密说明

AES（高级加密标准）是目前最安全、最广泛使用的对称加密算法之一。支持128、192和256位密钥长度，具有极高的安全性和加密效率。适用于文件加密、数据传输加密等场景。

📘 DES加密说明

DES（数据加密标准）是一种经典的对称加密算法，使用56位密钥。虽然现在被认为安全性较低，但在某些旧系统中仍在使用。建议在非敏感数据场景下使用，或考虑升级到3DES或AES。

📘 3DES加密说明

3DES（三重数据加密算法）是DES的增强版本，对数据进行三次DES加密，大大提高了安全性。使用168位密钥（实际有效密钥长度112位），是DES的安全替代方案，但速度较慢。

📘 RC4加密说明

RC4是一种流加密算法，以其简单性和速度著称。曾广泛应用于SSL/TLS和WEP等协议中。但由于发现了一些安全漏洞，现已不推荐用于新系统。可用于非关键数据的快速加密。

📘 Rabbit加密说明

Rabbit是一种高速流加密算法，由欧洲eSTREAM项目选定。具有优秀的性能和安全性，加密速度非常快。适用于需要高性能加密的场景，如实时通信、视频流加密等。

📘 RSA非对称加密说明

RSA是一种非对称加密算法，使用公钥加密、私钥解密（或私钥签名、公钥验证）。密钥长度支持512、1024、2048、4096位。适用于数字签名、密钥交换、身份认证等场景。注意：RSA加密有数据长度限制，通常用于加密对称密钥或小数据。

📘 HMAC签名说明

HMAC（Hash-based Message Authentication Code）是一种基于哈希函数的消息认证码算法。用于验证消息的完整性和真实性，常用于API签名、消息验证等场景。支持MD5、SHA-1、SHA-256、SHA-512等哈希算法。
